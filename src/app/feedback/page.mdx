# Things you can report

- Feature Requests
- Bugs
- Corrections to individual unit data
- Anything that should get my attention!

# Things to include in a report

- For **Bugs**, include:
  * Include when you started noticing the bug
  * How to reproduce it, if it only happens on certain devices, etc
  * Try to include screenshots, or better, a video of you reproducing the bug
- For **Feature Requests**, include:
  * Why this feature should be added (what problem is it solving, etc)
  * How your suggestion solves the problem
  * Optional: a drawing or model of what it could look like
- For **Corrections to Individual Unit Data**, include:
  * In game screenshots of **all** the values that need to be updated

# How to actually report

Ok, so now where do you even type this stuff out? Well you are free to contact me and submit your reports through any of these methods (only use one, unless I don't respond in one of them):

- [GitHub issues](https://github.com/agokule/cow-calc/issues)
- Pinging me in the [Supremacy Discord Server](https://discord.com/invite/Tuqxm7yTec) or DMing me on Discord. The username is "@superheroyrr"
  * If you are going to DM, make sure that your first sentence says it is about Cow Calc, or else I might think it is spam and might block you.
  * Pinging me in the server is probably best if it is a quick small issue

# Contributing

This project is open source on [GitHub](https://github.com/agokule/cow-calc/) if you didn't know! This means that anyone who knows how to code can implement their own feature requests, bug fixes, etc on their own!

Please do consider doing this if you know programming, especially for correcting unit data, since that should would be the easiest type of contribution to make.
